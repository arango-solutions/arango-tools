IMPORTANT — read before exposing this:
  * Single-tenant: runs as exactly ONE replica. Connection state is in-memory
    and shared by everyone who can reach it. Deploy one release per team.
  * No application-level authentication yet. Keep the Service private (the
    default is ClusterIP) and/or front it with your own auth proxy + TLS.
  * The pod can run the ArangoDB client tools and read its own filesystem.
    It runs as a non-root user with a read-only root filesystem.
